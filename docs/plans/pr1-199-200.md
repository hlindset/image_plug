## #200 — extend EAST/SOUTH offset sign + position clamp (localized, low risk)

**File:** `lib/image_pipe/transform/operation/extend_canvas.ex`, `offset/5` (line 154).

**Current:** `base_offset(...) + round(configured_offset)` for *all* anchors, no clamp.
- west/north base 0, center base = center_origin, right/bottom base = `canvas − image`.

**Upstream (imgproxy `calc_position.go`, allowOverflow=false):**
- west: `left = offX`; north: `top = offY` → ADD (already correct).
- center: base + offset → ADD (already correct).
- east: `left = outer − inner − offX` (`:44-46`); south: `top = outer − inner − offY` (`:48-50`) → SUBTRACT.
- All anchors then clamp left/top to `[0, outer − inner]`.

**Fix:**
1. Negate the offset for right (`:x`) / bottom (`:y`) anchors only.
2. Clamp the resolved origin to `[0, canvas_size − image_size]` (range always valid:
   ExtendCanvas guarantees `canvas ≥ image`).

```elixir
defp offset(axis, gravity, configured_offset, image_size, canvas_size) do
  base = base_offset(axis, gravity, image_size, canvas_size)
  signed = base + offset_direction(axis, gravity) * round(configured_offset)
  signed |> max(0) |> min(canvas_size - image_size)
end

defp offset_direction(:x, {:anchor, :right, _y}), do: -1
defp offset_direction(:y, {:anchor, _x, :bottom}), do: -1
defp offset_direction(_axis, _gravity), do: 1
```

**Verification of the #200 fixture** (`extend_offset_east_marker`, `rs:fit:400:150/ex:1:ea:20`,
canvas 400, image 200): base 200, signed `200 − 20 = 180`, clamp → 180. imgproxy left=180,
image right edge at x=379. ✅

**No-regression reasoning for existing extend constellations:** west/center cases keep ADD;
south/center cases with offset 0 are unchanged (`base + (±1)·0`, clamp no-op since
base ∈ [0, canvas−image]). Only east/south *with a nonzero offset* changes.

## #199 — fit+dpr single-round fold (broad-blast-radius care)

**File:** `lib/image_pipe/transform/operation/resize.ex`, base derivation + `apply_dpr`.

**Root cause:** the fit-derived (non-binding) axis is rounded in `fit_inside`/`requested_box`/
`apply_zoom` (dpr=1 box), then rounded *again* in `apply_dpr` (× dpr). imgproxy folds the
fit/zoom/dpr scale into ONE `imath.Scale` and rounds once (`prepare.go` calcScale/calcSizes).
`rs:fit:300:200/dpr:2` on 1600×1200: ImagePipe `round(266.67)=267 ×2 → 534`; imgproxy
`round(266.67×2)=533`.

**Fix:** match imgproxy's model exactly — `round(source_dim × single_scale)` per axis, where the
single scale folds the fit/fill ratio, zoom, and dpr. Implement by deferring rounding in the
**pre-dpr base derivation** (carry floats) and rounding ONCE in `apply_dpr`. Concretely, drop
`positive_round` from exactly three pre-dpr helpers: `fit_inside`, the two `:auto`-axis clauses
of `requested_box`, and `apply_zoom`. `apply_dpr` keeps the single `positive_round` (which also
restores the `max(1)` floor at the end).

**Corrected blast radius (review BLOCKER-1):** this is NOT a "dpr=1 bit-identical" change. For a
fit/fill request with one `:auto` axis, deferring the `requested_box` round changes the ratio fed
to `fit_inside`'s binding selection, so the *derived* axis can shift by 1px even at dpr=1 — e.g.
`fit:auto:200` on 100×109 goes 199→**200**. That shift is a **correctness improvement**: imgproxy
applies one scale to the source (`round(109×200/109)=200`), so the old 199 was a latent
double-round divergence; the fold fixes it as part of #199's "one round" intent. Both-axes-given
fit/fill (e.g. `fill:W:H`) is unaffected (no `:auto` derivation). The binding axis is unaffected
(`round(req_bind × dpr)` is exact either way).

**Downstream stays rounded (terminal rounds):** `cover_resize_dimensions`, `target_box_dimensions`,
`scale_dimensions`, `clamp_to_source`, and `scaled_min` run on the already-rounded post-`apply_dpr`
`requested` and are the terminal round for the fill intermediate / min path — they keep
`positive_round`. `result_crop_box` already single-rounds `value·zoom·dpr` (imgproxy-correct,
unchanged). `effective_dpr` consumes the float base as a ratio (robust); its `:auto`/`dpr: 1.0`
clauses still match.

**Acknowledged secondary changes (deliberate, imgproxy-correct):** the deferred `max(1)` floor
(sub-pixel derived axes now floor once at the end) and folded zoom rounding both move toward
imgproxy's single-scale model. None are covered by an existing fixture; the full differential +
unit suites are the regression net (a previously-passing fixture shifting would mean the fold is
wrong, not that the fixture was right).
